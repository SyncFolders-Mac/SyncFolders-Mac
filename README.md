## 📖 Overview

<a href="#syncFolders">SyncFolders</a> serves Mac users who need reliable folder-level backup and synchronization beyond what Time Machine's full-system backup approach provides — professionals backing up specific project folders, users maintaining synchronized folders across multiple drives, and anyone needing more granular control over what gets backed up and how.

The <a href="#syncFolders">project folder backup use case</a> applies SyncFolders to specific important project directories — keeping work folders backed up to an external drive on a regular schedule without the overhead of full-system backup, useful for users who want targeted backup of specific important data rather than comprehensive system backup. The <a href="#syncFolders">multi-computer folder sync use case</a> uses two-way synchronization to keep a folder consistent between a Mac and another computer or external drive used across multiple machines, useful for users who work from multiple locations or computers and need consistent access to the same files.

The <a href="#syncFolders">network and NAS backup use case</a> applies SyncFolders' network sync support for backing up to network attached storage devices common in home and small office setups, providing automated synchronization to NAS backup destinations on a regular schedule. <a href="#syncFolders">Scheduled reliability for non-technical users</a> makes SyncFolders valuable specifically because the automated scheduling removes the dependency on remembering to manually back up — once configured, synchronization happens consistently without ongoing manual attention.</p>

<a href="#syncFolders">Granular folder-level control</a> distinguishes SyncFolders from full-system backup approaches — rather than backing up an entire Mac, SyncFolders focuses specifically on designated folders, providing precision for users who know exactly which data matters most and don't need comprehensive system-level backup. <a href="#syncFolders">Conflict-aware bidirectional sync</a> handles the genuine complexity of two-way synchronization correctly — many simpler sync tools handle only one-way backup safely, while SyncFolders' conflict resolution makes bidirectional sync between actively used locations reliable.

<a href="#syncFolders">Lightweight resource usage</a> runs synchronization operations efficiently without significant impact on Mac performance during scheduled sync runs, particularly important for synchronization that happens automatically in the background without user awareness. <a href="#syncFolders">Regular macOS compatibility maintenance</a> keeps SyncFolders functioning correctly across macOS updates and changing file system behaviors.</p>

The conflict resolution capability is what distinguishes reliable bidirectional synchronization from simpler tools that only handle one-way backup safely. True two-way sync between actively used folder locations requires handling the case where the same file changes in both locations between sync runs — a genuinely complex scenario that naive synchronization approaches handle poorly, either by silently overwriting changes or by failing unpredictably.
